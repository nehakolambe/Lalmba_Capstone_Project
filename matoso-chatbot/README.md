# Matoso Chatbot Frontend

React interface for the Matoso / Mama Akinyi assistant. It handles authentication, registration, chat UI, and progress tracking against the Flask backend (see `../backend`).

## Running Locally
```powershell
cd matoso-chatbot
npm install
npm start
```
The app runs on [http://localhost:3000](http://localhost:3000) and expects the backend at `http://localhost:5000` (or the value of `REACT_APP_API_BASE`).

## Creating a New User
1. Start both the backend and this React app.
2. On the login screen, click **Create one here**.
3. Enter your full name, the username you want to reserve, a 4+ digit PIN/password, and any additional details that may help support you.
4. Submit the form. On success you will see a brief confirmation and be automatically signed in.
5. If the username is already in use or the backend validation fails, the error message will be displayed under the form.

## Checking Mama Akinyi's AI status
- Once logged in, use the **Check AI status** button to call the backend `/ollama/health` endpoint. The UI will confirm whether Ollama is reachable and list the available models.
- If you see "Mama Akinyi is offline," open a new terminal window and run `ollama run llama2` (or your preferred model), then press **Check AI status** again.
- A "Cannot reach the Matoso server" message usually means the Flask backend is not running on `http://localhost:5000`.
